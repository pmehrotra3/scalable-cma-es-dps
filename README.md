## Overview

Evolutionary strategies (ES) have been fairly limited in the RL community due to their lack of competitiveness against Policy Gradient Search (PGS) algorithms, primarily because of high sample and computational inefficiency. This repository accompanies the thesis that explores Direct Policy Search using variants of Covariance Matrix Adaptation Evolution Strategy (CMA-ES) — specifically sep-CMA-ES and two forms of Blockwise CMA-ES (Simultaneous and Sequential) — and benchmarks them against standard PGS methods including PPO, TRPO, A2C, DDPG, SAC, TD3, and TQC on high-dimensional continuous control tasks.

## Repository Structure

```
scalable-cma-es-dps/

│   ├── train_cma_direct_policy_search/

│   ├── train_simultaneous_blockwise_cma_direct_policy_search/
│   ├── train_sequential_blockwise_cma_direct_policy_search/
│   ├── train_a2c.py
│   ├── train_ppo.py
│   ├── train_sac.py
│   ├── train_ddpg.py
│   ├── train_td3.py
│   ├── train_tqc.py
│   ├── train_trpo.py
│   └── run_all.sh
├── output/                          # Experiment results
├── visualise.py                     # Visualisation
├── requirements.txt
└── LICENSE
```

### `Agents/`

Contains all training scripts. Each CMA-ES variant lives in its own subfolder (since it requires a compiled C++ pybind11 neural network module). The SB3 baselines (`train_*.py`) live directly in `Agents/` and use Stable-Baselines3.

### `run_all.sh`

Orchestrates all experiments. CMA-ES experiments run first and complete fully before SB3 baselines begin. Within each group, non-Humanoid environments run with `MAX_PARALLEL=4` concurrent jobs; Humanoid-v5 runs sequentially (`MAX_PARALLEL=1`) due to its high memory cost. Progress is logged to stdout with timestamps.

### `output/`

Auto-generated by the training scripts. Organised as `output/<env_name>/<algorithm_name>/`, containing per-run CSV logs:
- `episode_log_run_N.csv` — per-episode return and length
- `system_log_run_N.csv` — per-episode wall time, CPU time, RAM, and CPU %

### `visualise.py`

Reads all CSVs in `output/` and generates three files per environment:
- `plot_reward_vs_timestep.pdf` — 100-episode moving average reward vs. environment steps
- `plot_reward_vs_episode.pdf` — 100-episode moving average reward vs. episode number
- `summary_table.pdf` — per-algorithm summary table (avg reward, best reward, avg CPU %, total CPU time, total wall time, avg RAM)

For full details on what is generated and how metrics are computed, see Section 7 (Experiment Design) of the thesis.

## Installation

### 1. Prerequisites

- Python 3.10+
- macOS ARM64 (pre-compiled C++ module provided); other platforms require recompilation (see below)
- [`swig`](https://www.swig.org/) — required for `box2d-py`: `brew install swig`

### 2. Install Python dependencies

```bash
pip install -r requirements.txt
```

### 3. C++ Neural Network Module (CMA-ES variants only)

The pre-compiled `nn` pybind11 module (`nn.cpython-*.so`) is provided for **macOS ARM64** and is already present in each CMA-ES agent subfolder. No recompilation is needed on macOS ARM64.

For other platforms, recompile inside each CMA-ES agent folder. Note that compiler flags differ by platform — the command below is for macOS ARM64 only and will need to be adapted for Linux or Windows.

**macOS ARM64:**
```bash
cd Agents/<cma_agent_folder>
c++ -O3 -shared -fPIC \
    $(python3 -m pybind11 --includes) \
    $(python3-config --ldflags) \
    bindings.cpp \
    -undefined dynamic_lookup \
    -o nn$(python3-config --extension-suffix)
```

## Running Experiments

All commands should be run from the `scalable-cma-es-dps/` root directory.

### Run all experiments

```bash
cd Agents && bash run_all.sh
```

CMA-ES experiments run first, SB3 baselines second. A training log is printed directly to stdout. To save it to a file:

```bash
cd Agents && bash run_all.sh 2>&1 | tee ../training.log
```

### Run a single algorithm

```bash
# CMA-ES variant (must cd into subfolder)
cd Agents/train_sep_cma_direct_policy_search
python3 train_sep_cma_direct_policy_search.py HalfCheetah-v5 --total-timesteps 1000000 --num-runs 5

# SB3 baseline (run from Agents/)
cd Agents
python3 train_ppo.py HalfCheetah-v5 --total-timesteps 1000000 --num-runs 5
```

### Generate plots and tables from training data

```bash
python3 visualise.py
```
